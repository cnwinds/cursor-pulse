# 钉钉机器人使用指南

小脉支持 **自然语言对话**。直接用口语描述你想做的事即可，例如「帮我绑定 Cursor Key」「这个月花了多少」「有哪些功能」。

下文各节的命令示例仅供参考，**不是固定口令**。Agent 会根据你当前可用的 tools 选择并调用对应能力。

询问「帮助」「能做什么」「有哪些功能」时，机器人会根据 **当前可见的技能卡片**（SkillRegistry）动态总结用法，详细说明按需从 Skill 文档加载。

技能说明源文件：`assistant_platform/skills/docs/`（按服务域分组，非一命令一技能）。

管理后台「技能一览」可预览上述技能文档与人设 Prompt，**不可在线编辑**；修改请改仓库并发版。

## 使用前须知

### 账期

- 未写明账期时，默认指 **当前统计账期**（通常为当月，格式 `YYYY-MM`，如 `2026-07`）。
- 部分命令支持指定账期，例如：`报告 2026-06`、`聚合 2026-06`。

### 记录 ID 前缀

- 摄取、试用申请、Key 借用等记录均有唯一 ID。
- 在命令中只需填写 **ID 的前 8 位字符**（系统会匹配唯一记录），例如：`确认 a1b2c3d4`、`审批 通过 e5f6g7h8`。

### 使用场景（私聊 / 群聊）

| 场景 | 建议方式 | 说明 |
|------|----------|------|
| 绑定 / 解绑 Cursor Key | **私聊** | Key 属于敏感凭证，请勿在群内发送 |
| 借 Key / 归还 Key | **私聊** | 回复中会包含用户侧 Key（自助为 `pka_` 别名） |
| 上报非 Cursor 用量 | **私聊优先** | 群内提交会极简确认，完整摘要通过私聊发送 |
| 分享心得 | 私聊或群内均可 | 收录进团队知识库 |
| 发布月报 | 管理员在群内操作 | 机器人将月报发到配置的目标群 |

### 权限角色

| 角色 | 判定方式 | 典型能力 |
|------|----------|----------|
| **普通成员** | 默认 | 查本人提交、绑定 Key、上报用量、借 Key 等 |
| **主管** | 成员档案中的直属主管 | 审批下属的「试用申请」 |
| **管理员** | 台账 `owner` / `operator`，或配置中的钉钉 admin userid | 月报、聚合、成员管理、告警、借用列表等 |

---

### 绑定工作群（管理员）

将机器人拉入目标群后，在群内 **@ 机器人** 并发送：

```
启动
```

同义：`激活`、`开始`、`开工`、`start`

**作用：**
- 将当前群设为 **工作群**（月报、催办等群消息发到此群）
- 配置写入管理后台「钉钉集成」，并持久化到团队设置
- 机器人会在群里打招呼，说明群内与私聊可做的事

**权限：**
- 首次绑定：群内任意成员可发送「启动」
- 更换工作群：仅管理员（台账 owner/operator 或配置的钉钉 admin）

## 一、Cursor 用量（自动同步）

> **重要：** Cursor 用量 **不需要** 手工上报、上传 CSV 或发截图。绑定 User API Key 后，系统每日自动同步。

### 绑定 Cursor API Key

**命令格式：**

```
绑定 cursor key crsr_xxxxxxxx
```

若你名下有 **多个** Cursor 账号，需同时指定邮箱：

```
绑定 cursor 你的邮箱@example.com key crsr_xxxxxxxx
```

**说明：**

- `crsr_` 开头的是 Cursor **User API Key**，在 Cursor → Settings → Integrations 中创建。
- 仅 **账号主使用人** 或 **管理员** 可为该账号绑定 Key。
- 绑定成功后立即尝试同步，并回复写入的事件条数。
- 若 Key 对应邮箱与台账账号不一致，会提示校验失败，请核对后重新发送。

### 解绑 Cursor API Key

**命令格式：**

```
解绑 cursor
```

或指定邮箱（多账号时必填）：

```
解绑 cursor 你的邮箱@example.com
```

**说明：**

- 解绑后停止该账号的自动同步；历史已入库数据保留。
- 权限要求与绑定相同。

### 查看本人 Cursor 额度

**命令：**

```
额度
```

（同义：`我的额度`）

**说明：**

- 展示你名下 Cursor 账号的当前计费周期额度使用情况（充足 / 偏紧 / 已耗尽等）。
- 若尚未绑定 Key 或无额度快照，会提示暂无数据或需先绑定。

## 二、非 Cursor 工具用量（手工上报）

适用于：**智谱、MiniMax、Codex** 等未接入 API 自动同步的工具。

### 文本上报

**命令格式：**

```
上报 <工具名> <数值> [单位]
```

**示例：**

| 命令 | 含义 |
|------|------|
| `上报 智谱 85` | 智谱用量 85（按账号套餐默认单位） |
| `用量 minimax 12000 calls` | MiniMax 调用 12000 次 |
| `上报 codex 45 messages` | Codex 消息 45 条 |

**支持的工具别名：**

| 工具 | 可识别的名称 |
|------|----------------|
| 智谱 | `智谱`、`zhipu`、`glm`、`清言` |
| MiniMax | `minimax`、`海螺` |
| Codex | `codex`、`chatgpt`、`openai`、`gpt` |

**说明：**

- 你须已在台账中拥有对应工具的账号，且主使用人为你本人；否则请联系管理员开户。
- 提交后 **直接入库** 计入统计，无需管理员审核。
- **不要** 用此方式上报 Cursor 用量。

### 截图自动识别

向机器人 **私聊发送** 智谱 / MiniMax / Codex 控制台的用量截图（图片消息），系统会尝试 OCR 识别并 **直接入库**。

- 识别置信度不足时，会提示改用手工文本上报（如 `上报 智谱 85`）。
- 若你名下有多个不同工具的账号，截图无法自动判断归属时会提示你指定工具或改用手工文本上报。

### CSV / Excel 文件

私聊发送非 Cursor 工具的用量 **CSV 或 Excel（.xlsx）** 文件（亦可粘贴 CSV 文本）。

- Cursor 的 usage-events CSV **不再需要**；请改绑 API Key。

## 三、提交状态与用量查询

### 查看团队提交进度（管理员）

**命令：**

```
状态
```

**说明：**

- 展示当前账期各 **active 成员** 是否已完成用量提交（✅ 已提交 / ❌ 未提交）。
- **仅管理员** 可查看。

### 查看本人用量明细

**命令：**

```
我的用量
```

**说明：**

- 展示你名下各 Cursor 账号的用量合计，以及 **按模型** 拆分的明细表（次数、Tokens、费用、占比）；无总览表。
- 默认按记账周期统计；可加「自然月」或 `YYYY-MM` 指定账期，如 `我的用量 自然月`。
- 若正在 **借用 Key**：有 Proxy 精确计量时展示次数 / Tokens / 估算费用与模型表；否则回退已消耗 % / 还能用 %。不展示借出人。
- 需已绑定 Cursor Key 或有进行中的借用。

### 查看本人提交记录

**命令：**

```
我的
```

**说明：**

- 展示你在当前账期的最近一次提交时间及渠道（如自动同步、钉钉手工上报等）。
- 若尚无记录，会明确提示。

### 自然语言查询（大模型）

**命令格式：**

```
查询 <问题>
```

（同义前缀：`问`；也可直接发送自然语言，如 `谁用得最多`）

**示例：**

- `查询 谁 tokens 最多` — 团队排名类问题（需管理员权限）
- `查询 我本月用了多少` — 本人用量详情

**说明：**

- 由助手 **大模型** 理解你的问题，并基于已入库数据作答（不再使用固定关键词匹配）。
- 需在后台启用 `assistant_llm`；未启用时请用「额度」「我的」等固定命令。
- 回答基于已同步或已确认入库的数据；未确认的手工提交不计入。

## 四、试用申请与审批

### 申请 Cursor 账号（开户）

**命令格式：**

```
申请 Cursor
```

或附带理由：

```
申请 Cursor 新项目开发需要
```

**说明：**

- 向团队申请分配一个 **Cursor 账号台账**（开户），与「绑定 Key」「借 Key」是 **不同流程**。
- 提交后进入待主管审批；若档案中配置了直属主管，会通知主管。
- 审批通过并分配账号后，你仍需自行执行 **`绑定 cursor key ...`** 才能自动同步用量。

### 主管 / 管理员审批

**命令格式：**

```
审批 通过 <申请ID前8位>
审批 拒绝 <申请ID前8位>
```

**说明：**

- **直属主管** 可审批其下属的申请；**管理员** 可审批任意申请。
- 管理员审批通过时，若存在空闲试用账号会自动分配；否则提示需在后台手动分配。

## 五、临时 Key 借用

当 **你名下 Cursor 账号额度不足**（偏紧或已耗尽）时，可从团队富余账号自助借用临时 User API Key。

> 与「申请开户」「绑定本人 Key」无关；借用的是他人账号的临时 Key。

### 借 Key

**命令格式（建议私聊）：**

```
借 Key
```

或附带理由：

```
借 Key 本周赶版本额度不够
```

**同义说法：** `借key`、`申请临时key`、`续借key` 等。

**前置条件：**

1. 你已有分配的 Cursor 账号，且已绑定自己的 Key（系统需能判断你确实额度不足）。
2. 当前 **没有** 进行中的借用（有则需先归还）。
3. 团队中存在可借出的富余账号。

**成功时：**

- 下发的是 **代理别名 Key（`pka_` 开头）**，不是 Cursor 官方 Key。
- **必须** 配置 `HTTPS_PROXY` 指向团队代理后再用 `CURSOR_API_KEY=pka_…` 登录；代理会在服务端换成绑定的 Cursor Key。
- 可发送「我的借用」再次查看该 `pka_` Key。

**失败时常见原因：**

- 额度尚充足 → 不支持自助借 Key。
- 尚无 Cursor 账号 → 请先 `申请 Cursor`。
- 无富余账号 → 联系管理员。

### 查看当前借用

**命令（任选）：**

```
我的借用
我借的key
借用状态
```

**说明：**

- 展示创建时间、用量摘要（有 Proxy 时为精确次数 / Tokens / 估算费用，否则为近似消耗）、是否在计费重置日自动回收，以及 **完整用户侧 Key**（自助借用为 `pka_`）。不展示借出人。
- 请勿在群内发送此类命令（回复含 Key）。

### 归还 Key

**命令：**

```
归还 Key
```

（同义：`归还借用`、`归还key`）

**说明：**

- 主动结束借用，远程撤销临时 Key 并结算近似消耗。

### 管理员：借用列表与撤销

| 命令 | 权限 | 说明 |
|------|------|------|
| `借用列表` 或 `借用` | 管理员 | 列出团队活跃借用（最多 15 条），含 ID 前缀、借用人、来源账号、近似消耗 |
| `撤销借用 <ID前8位>` | 管理员 | 强制结束指定借用并撤销远程 Key |

### 设置借 Key / 绑 Key 引导图

**命令：**

```
设置引导图
```

**说明：**

- **管理员** 专用。发送该命令后，**下一条消息** 须为图片，用于更新绑定 Key、借 Key 流程的配图说明。
- 普通成员绑定或借 Key 失败时，机器人可能自动附上该引导图。

## 七、管理员命令

以下命令均需 **管理员** 权限；无权限时回复「无权限」。

| 命令 | 格式 | 作用 |
|------|------|------|
| 生成月报 | `报告 [账期]` | 生成当月团队用量月报并 **发布到配置的目标钉钉群** |
| 重新聚合 | `聚合 [账期]` | 按账期重算团队用量汇总（事件数、Tokens、费用等） |
| 成员名单 | `成员` | 列出 active 催办成员及钉钉 userid |
| 添加成员 | `成员 添加 <userid> <姓名>` | 将成员加入 active 名单（参与提交进度统计与催办） |
| 移除成员 | `成员 移除 <userid>` | 将成员设为 inactive |
| 异常检测 | `告警 [账期]` | 运行用量异常规则并输出告警列表 |
| 导出用量 | `导出 [账期]` | 导出账期用量 CSV 到服务器本地目录 |

## 九、常见问题

**Q：Cursor 还要每月上传 CSV 吗？**  
A：不需要。绑定 User API Key 后自动每日同步。

**Q：「申请 Cursor」「绑定 Key」「借 Key」有什么区别？**  
A：

1. **申请 Cursor** — 向团队要一个 Cursor **账号名额**（台账开户）。
2. **绑定 Key** — 把你自己的 Cursor **User API Key** 绑到已分配账号上，用于 **自动同步你自己的用量**。
3. **借 Key** — 你名下账号 **额度不够用** 时，临时借用团队里 **别人富余账号** 的额度；自助下发的是 `pka_` 代理别名 Key，须走团队代理。

**Q：发送了命令但提示「未知命令」？**  
A：检查拼写与空格；发送 `帮助` 查看摘要。自然语言对话（非上述固定命令格式）由助手理解后路由，不保证与命令完全一致。

**Q：手工上报后多久能查到？**  
A：文本与截图识别成功后立即入库；发送 `我的` 可查看提交记录，Web 后台「摄取记录」可查看主指标。
